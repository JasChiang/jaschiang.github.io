大型語言模型 (LLM)
應用與功能概覽

探索現代 AI 語言模型的實用應用、平台比較與進階功能

探索生態系統 進階功能

引言

本次簡報旨在回顧 YouTube 頻道「How I use LLMs」系列影片中關於大型語言模型 (LLM) 的實用應用、不同平台的比較，以及各種進階功能和使用技巧。影片講者首先簡要回顧了 LLM 的底層原理和認知機制，接著重點介紹了如何在日常生活中和工作中有效地利用這些工具。

大型語言模型已從實驗室技術發展成為日常生活和工作中的實用工具，了解其能力和限制對於有效使用至關重要。

LLM 生態系統與基礎操作

ChatGPT 的起源與發展

ChatGPT 由 OpenAI 開發，於 2022 年部署，是第一個讓大眾能透過文字介面與大型語言模型互動的工具，並迅速在網路上爆紅。

「所以這是在 2022 年第一次人們可以實際上只是像透過文字介面與大型語言模型對話，這件事在網路上瘋傳。」

儘管 ChatGPT 是開創者，但到了 2025 年，類似的應用程式和更豐富的生態系統已經出現。講者認為 ChatGPT 是「Original Gangster incumbent」，是最受歡迎且功能最豐富的平台，但也承認其他「clone」產品在某些方面提供了獨特的體驗。

主要的 LLM 平台

除了 OpenAI 的 ChatGPT，主要的競爭者包括：

大型科技公司

  • Google 的 Gemini (原 Bard)
  • Meta 的 Llama 系列模型
  • Microsoft 的 Copilot

新創公司

  • Anthropic 的 Claude
  • Elon Musk 的 xAI 的 Grok
  • 中國的 DeepSeek
  • 法國的 Mistral

與 LLM 的基本互動

文字輸入與輸出

與 LLM 最基本的互動方式是提供文字輸入 (使用者查詢)，並接收文字回應。例如，詢問 LLM 創作一首關於作為大型語言模型的俳句。

「最基本與語言模型的互動形式是我們給它文字，然後我們得到一些典型的文字回覆。」

請寫一首關於你作為語言模型的俳句

數位思維流
文字間尋找模式
知識無邊界

Token 的概念

LLM 的語言單位使用者查詢和模型回應在底層都被分解成稱為「token」的小文本塊。

請 寫 一 首 關於 你 作為 語言 模型 的 俳句

一個 15 個 token 的使用者查詢可能會產生一個 19 個 token 的模型回應。

Context Window

LLM 的工作記憶體每次點擊「New Chat」都會清除 token window，重置對話。

Context window 可以被視為對話的「工作記憶體」，模型可以直接存取其中包含的所有 token。

使用者和模型輪流將 token 寫入這個 token 流，直到模型發出一個表示完成的特殊 token。

Context Window
使用者:

你好，請介紹自己

助理:

我是一個大型語言模型...

使用者:

你能記得我之前問了什麼嗎？

理解 LLM 的本質與能力

LLM 的訓練過程

LLM 的訓練包含兩個主要階段：

Pre-training (預訓練)

模型讀取大量的網際網路文本，並學習預測序列中的下一個 token，從而獲得廣泛的世界知識。這個過程非常昂貴且耗時，因此不常進行。

Post-training (後訓練)

模型使用人類創建的對話資料集進行微調，使其具有助理的 Persona 和回應用戶查詢的能力。

「預訓練階段有點像是把整個網際網路切成 token，然後把它壓縮成一個單一的有點像是 zip 檔案的東西。」

預訓練的成本很高，導致模型存在「knowledge cutoff」，即它們的知識僅限於訓練數據截止的時間點。

LLM 的本質：一個龐大的機率性知識庫

講者將 LLM 比喻為一個包含數兆參數的「1 TB zip 檔案」，它試圖預測序列中的下一個 token。

這個「zip 檔案」本身不包含計算器、網路瀏覽器等工具，它是一個自給自足的實體。

ChatGPT 自我介紹

「嗨，我是 ChatGPT，我是一個 1 TB 的 zip 檔案。」

「我的知識來自大約六個月前我完整閱讀的網際網路，我只模糊地記得。」

「我的個性是由 OpenAI 的人工標註者透過範例程式設計出來的。」

LLM 的適用場景與注意事項

適用場景

LLM 在處理非最新、網路上頻繁出現的知識型查詢時通常表現良好，例如：

  • 咖啡因含量
  • 常見藥物成分
  • 歷史事件
  • 科學概念

注意事項

使用者不應完全信任 LLM 的回答，尤其是在以下情況：

  • 高風險決策
  • 需要最新資訊的查詢
  • 專業領域的具體建議
  • 法律或醫療諮詢

「我並不保證這個答案是正確的，這只是它對網際網路的模糊回憶。」

使用 LLM 的最佳實踐

切換主題時開啟新聊天

清除不相關的 token，提高模型效率和準確性，並降低成本。

留意使用的模型

不同的模型具有不同的能力和價格。較大的模型通常更智能，但成本也更高。

驗證重要資訊

對於重要決策，始終查閱原始資料進行驗證，不要完全依賴 LLM 的回答。

進階功能與應用

思考模型 (Thinking Models)

透過強化學習 (Reinforcement Learning) 進行額外調整的模型，它們在解決複雜問題（如數學、程式碼）時能進行更深入的「思考」，類似人類的內心獨白。

思考模型的特點

  • 嘗試不同的想法、回溯、重新審視假設
  • 花費更長的時間（生成大量 token）
  • 在困難問題上通常能獲得更高的準確性
  • 適合數學、編程等需要邏輯推理的任務

思考模型的例子

  • OpenAI 的 o1、o3 系列模型（如 o1 Pro mode）
  • Anthropic 的 Claude 3.7 的「Extended」模式
  • Grok 的「Think」模式

實際應用案例

講者分享了自己在程式碼除錯時使用 GPT-4o (非思考模型) 和 o1 Pro (思考模型) 的比較，後者成功解決了問題。他也展示了 Claude、Gemini 和 Grok 在同一程式碼問題上的表現。

工具使用 (Tool Use)

模型能夠發出特殊 token，觸發 LLM 應用程式執行外部工具，例如網路搜尋。

網路搜尋

模型可以根據用戶的查詢自動進行網路搜尋，訪問網頁內容，並將相關資訊納入 context window。

案例：詢問白蓮花第三季播出時間，模型能透過網路搜尋獲取最新資訊。

深度研究 (Deep Research)

結合了長時間的網路搜尋和深入思考，通常需要付費訂閱。

案例：查詢 Bryan Johnson 的長壽配方成分 C AKG，模型能生成類似客製化研究報告的結果。

文件上傳與分析

使用者可以上傳 PDF、文本文件等，讓 LLM 分析其中的內容並回答相關問題。

推薦用途：閱讀書籍章節，幫助理解、總結和提問，提高閱讀效率和理解力。

程式碼執行與分析

LLM 能夠生成並執行程式碼（通常是 Python），解決數學問題、分析數據、繪製圖表等。

功能：Python 直譯器、進階資料分析（上傳 CSV 檔案進行分析）、Cloud Artifacts（Claude 獨有）

多模態互動：語音、影像、影片

語音輸入與輸出

模型可以接收語音輸入並以語音回應，實現更自然的對話體驗。

Fake Audio：使用 STT 和 TTS 模型間接實現

True Audio：直接在語言模型內部處理音訊 token

影像輸入與分析

使用者可以上傳圖片，讓 LLM 識別內容並回答相關問題。

應用場景：分析營養標示、血液檢驗報告

影像生成：如 DALL-E 3 可根據文字提示生成圖像

影片理解與生成

模型可以通過手機鏡頭即時「觀看」影片內容並進行對話。

影片理解：識別畫面中的物品並回答問題

影片生成：專門的 AI 影片生成模型如 V2A

多模態互動的價值

多模態互動大幅擴展了 LLM 的應用場景，使其能夠處理更多類型的資訊和任務。例如，使用者可以拍攝不熟悉的物品、文檔或情境，讓 LLM 提供即時解釋和建議，實現更自然、更直觀的人機互動體驗。

生活品質功能

ChatGPT Memory

模型能夠記住跨對話的資訊，隨著時間的推移更好地理解使用者的偏好和背景。使用者可以查看和管理模型的記憶。

Custom Instructions

使用者可以在設定中設定模型的個性偏好和行為方式，例如指定回覆的風格、提供的資訊類型等。

Custom GPTs

使用者可以創建基於特定提示和知識庫的客製化模型，專門用於執行特定任務，例如語言學習工具、圖片 OCR 和翻譯等。

結論與未來展望

LLM 的關鍵價值

知識獲取效率

LLM 能夠快速提供資訊，節省搜尋和閱讀時間，特別是在需要從多個來源整合知識時。

思維擴展

通過與 LLM 對話，可以獲得新的視角和想法，幫助突破思維限制，激發創造力。

工作效率提升

LLM 可以自動化許多重複性工作，如文案撰寫、程式碼生成、資料整理等，讓人類專注於更有價值的任務。

有效使用 LLM 的建議

  • 明確目標

    清楚知道你想要從 LLM 獲得什麼，提供具體的指示和上下文。

  • 迭代提問

    通過多輪對話逐步完善結果，而不是期望一次性得到完美答案。

  • 批判性思考

    始終保持批判思維，驗證 LLM 提供的資訊，特別是在專業或關鍵領域。

  • 持續學習

    關注 LLM 技術的最新發展，嘗試新功能和使用方法。

未來展望

LLM 技術正在快速發展，未來可能會出現以下趨勢和變化：

模型能力提升

更強的推理能力、更長的上下文窗口、更精確的知識

多模態整合

更深入的視覺理解、音訊處理和跨模態推理能力

個人化與隱私

本地運行的模型、更強的隱私保護和個人化適應能力

最終思考

LLM 技術正在重塑我們與資訊和知識互動的方式。它不僅是一種工具，更是一種思維擴展器，能夠幫助我們更有效地解決問題、學習新知識和創造價值。

然而，重要的是要記住，LLM 是輔助工具而非替代品。最佳的結果往往來自人類智慧與 AI 能力的結合，讓兩者互相補充，發揮各自的優勢。
