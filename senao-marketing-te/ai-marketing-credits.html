關於本課程

本 AI 行銷課程網站是一個綜合性資源，旨在幫助行銷人員了解並應用 AI 工具來提升工作效率。

資料來源

本課程部分內容基於 Google Gemini Deep Research 所搜集的研究資訊

製作者

內容大綱經由 Jas Chiang 進行規劃，並針對行銷領域的 AI 初學者的可能需求進行調整。

製作工具與方法

開發環境

本網站使用 Windsurf 作為主要開發環境，在 Claude 3.7 Sonnet 的幫助下進行網頁設計。

使用技術

網站使用 HTML5、TailwindCSS 和 Font Awesome 圖標庫構建，確保視覺吸引力和響應式設計。

網頁製作提示詞來源

本網站的設計和內容生成使用了修改自 X 用戶 @op7418 的網頁生成提示詞，並根據 AI 行銷課程的特定需求進行了調整。

視覺化設計特色

流程圖設計

課程中的流程圖使用 HTML 和 CSS 創建，採用不同顏色區塊和圖標直觀展示各個步驟，並在移動設備上自動調整為垂直排列。

比較表格

比較表格採用卡片式設計，使用顏色對比和圖標增強視覺效果，幫助讀者快速理解不同概念或工具之間的差異。

關係圖

關係圖展示了不同概念、角色或步驟之間的連接和互動，使用不同顏色區塊和連接線清晰呈現複雜關係。

網頁生成提示詞

以下是用於生成本 AI 行銷課程網站的完整提示詞，修改自 X 用戶 @op7418 的網頁生成提示詞，並根據 AI 行銷課程的特定需求進行了調整：

內容要求

* 根據所提供的資料或文件（無論格式）中的內容來製作 HTML 網頁
* HTML 檔案名稱需與主題有關
* 所有頁面內容必須為繁體中文
* 保持原文件的重點資訊，但以更易讀、視覺化的方式呈現
* 網頁內容不能全部都是文字，必須有視覺元素等
* 文字內容超過300字的段落，必須考慮轉化為視覺圖表形式呈現
* 列表型資訊（步驟、比較、分類等）必須轉化為流程圖、比較表格或圖示卡片
* 每個主要頁面必須加入至少2-3個視覺化流程圖、比較表格或工作流程圖來呈現複雜概念
* 視覺元素必須使用 HTML、CSS 和 Font Awesome 圖標創建，確保響應式設計
* 當提供的資料內容過多（超過5000字或包含多個獨立主題）時，應拆分為多個HTML頁面
* 拆分頁面時，應建立清晰的導航結構，包括主頁面和子頁面之間的連結
* 拆分的頁面應遵循一致的設計風格和命名規則（如主題-子主題.html）
* HTML 網頁製作完成後，在 index.html 中新增連結與簡述
* 如果專案沒有 index.html，請新增 index.html
* 如果專案有 index.html，請在 index.html 中新增連結與簡述

設計風格

* 整體風格參考 Apple 官網的簡潔現代感設計
* 分析內容，根據內容建立層次結構，突出重要內容
* 配色方案若無指定，則以 Apple 官網為主
* 使用漸變色背景增加視覺吸引力
* 適當使用卡片式設計呈現相關內容組
* 確保視覺元素與文字內容相輔相成，增強理解
* 流程圖和工作流程圖應使用不同顏色區分不同步驟或階段
* 比較表格應使用清晰的視覺層次和顏色對比展示差異
* 文字密集段落應轉化為圖示卡片、時間軸或流程圖等視覺形式
* 多頁面專案應保持一致的頁眉、頁腳和導航元素，確保用戶體驗連貫

技術規範

* 使用HTML5、TailwindCSS 3.0+（通過CDN引入）和必要的JavaScript
* 程式碼結構清晰，包含適當註釋，易於理解和後續維護
* 使用 HTML 和 CSS 創建流程圖和比較表格，而非依賴外部庫
* 針對不同類型的資訊選擇最合適的視覺呈現方式（流程圖、比較表格、工作流程圖等）
* 使用 AOS (Animate On Scroll) 庫實現滾動動畫效果
* 確保所有視覺元素都有適當的 HTML 語義標記，提高可訪問性
* 文字轉圖表時，保留原始文字作為輔助說明或替代文本

RWD 響應式設計

* 頁面必須在所有裝置上（手機、平板、桌面）完美展示
* 針對不同螢幕尺寸調整版面佈局和字型大小
* 確保行動裝置也有良好的觸控體驗
* 使用 grid 和 flex 布局確保內容靈活適應不同螢幕
* 確保流程圖和比較表格在小螢幕上能自動調整為垂直排列
* 在移動設備上隱藏非必要的裝飾性元素，保持核心內容清晰可讀
* 複雜圖表在移動設備上可考慮分解為多個簡化版本

互動體驗

- 參考 Apple 官網增添互動效果提升用戶體驗
* 按鈕懸停時有輕微放大和顏色變化
* 卡片元素懸停時有精緻的陰影和邊框效果
* 頁面捲動時有平滑過渡效果
* 內容區塊加載時有優雅的淡入動畫
- 添加互動式元素
* 使用可折疊的問答區域（Accordion）處理長文本內容
* 添加標籤式內容切換（Tabs）組織相關主題
* 實現簡單的懸停提示（Tooltips）提供額外資訊
* 使用進度指示器顯示閱讀位置
